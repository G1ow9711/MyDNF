# 项目协作约定

- 禁止未经用户明确允许启动任何子智能体或并行 agent。只有用户针对当前任务明确批准后，才能启用；历史上的通用授权不视为当前授权。
- 获得当前任务授权后，能拆成独立模块、互不阻塞的开发、测试、审查、资料检索或验收任务时，才可使用多个 agent 并行执行。
- 并行 agent 的工作必须有清晰边界；不同 agent 不得同时修改同一文件。如必须修改同一文件，先由主 agent 合并计划后再执行。
- agent 产出的结论必须由主 agent 汇总、复核，并通过本项目测试、构建或浏览器验收后再提交。
- Git 提交信息必须使用中文。
- 当前原型优先打通全功能和战斗手感：人物、怪物建模可以先保持轻量，但攻击动作、受击动作、移动衔接、技能特效、怪物技能特效、打击停顿和反馈必须作为严格验收项。
- 角色与怪物模型可以简化，但技能命中帧、投射物飞行、动作取消、中断、动态目标重判和模型跟随动作必须由真实战斗状态驱动，不能只用静态贴图或纯装饰动画糊弄。
- 房间流、清怪、开门、进下一房等核心循环必须优先补真实键盘浏览器验收；静态 DOM、纯 reducer 测试只能作为辅助。
- 后续开发继续以 DNF 风格横版房间战斗为主线；新增系统或美术表现不能破坏键盘操作、严格命中帧、怪物攻击判定、存档和离线可玩性。
