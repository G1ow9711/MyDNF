# DNF 战斗控制、受击硬直与霸体设计

## 目标

补齐当前原型最明显的 DNF 战斗控制缺口：玩家可用 `↓+C` 做平滑后跳；普通杂兵被普通攻击命中后进入短硬直并中断正在蓄力的攻击；精英和 Boss 在护甲未破时保留霸体，受到伤害和护甲削减但不被普通攻击硬直、浮空、击倒或打断；破甲后恢复完整受击反应。

## 输入与玩家动作

- 命令缓存继续使用现有 700 ms 时限。缓存尾部严格匹配 `ArrowDown`, `KeyC` 时触发 `backstep`，并优先于 `KeyC` 的跳跃/快速起身映射。
- 单独 `KeyC` 保持现有跳跃；处于强受击快速起身窗口时仍保持快速起身。
- 后跳使用 180 ms 模型跟随位移，从当前位置缓动到反方向 74 px；朝向不改变。现有 420 ms 闪避、240 ms 无敌、260 ms动作锁和零资源消耗规则不变。
- 战斗按钮提示改为 `↓C`，使鼠标入口与真实键盘命令一致。

## 敌人受击规则

- 新增 `hitstunUntilMs`。普通杂兵在一次非致死普通命中后获得 280 ms 硬直；带 `stagger`、`control`、浮空或击倒的既有状态仍具有更高优先级。
- 硬直开始时清除当前怪物攻击状态，下一次攻击不得早于硬直结束。硬直期间 `beginEnemyAttack()` 不得重新起手。
- 精英/Boss 在命中前护甲大于零、未处于破甲窗口且本次命中不带 `guard-break` 时视为霸体。霸体只抑制普通硬直、浮空、击倒和攻击取消；伤害、护甲削减、击退数值、标记和命中特效仍正常结算。
- 护甲被普通命中削减到零时，本次命中仍按命中前霸体结算；下一次命中可造成硬直。`guard-break` 命中立即绕过霸体并开启现有 1800 ms 破甲窗口。
- 致死命中仍允许倒地/击败表现，不被霸体隐藏。
- 即时 `applyHit()` 与延迟命中效果必须调用同一受击反应解析器。

## 呈现与验收钩子

- 敌人根节点暴露 `data-enemy-hitstun-active`、`data-enemy-super-armor` 和 `data-armor-state="super-armor|broken|normal"`。
- 硬直状态使用独立 `hitstun` 模型动作，优先级低于浮空、击倒和控制，高于普通最近命中与攻击。
- 霸体使用稳定的金属轮廓和短促受击亮度反馈，不覆盖怪物正在执行的技能动作。
- 真实浏览器验收必须从挂载页面发送真实键盘事件：`ArrowDown` 后 `KeyC` 观察平滑反向位移；普通攻击观察杂兵硬直；攻击有护甲敌人观察霸体；破甲后观察可打断。

## 边界

本段不增加新职业技能、暴击系统、装备词条或新怪物模型。它只强化既有战斗控制和受击状态机，且不得破坏命中帧、hitstop、存档、难度倍率、房间门和现有 `KeyC` 快速起身。
