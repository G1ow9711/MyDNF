# 副本定向掉落与结算反馈设计

## 目标

让地下城的副本身份、难度和房间节点真实决定装备产出，并让玩家在房间结算及最终通关后立即看见具体奖励。保持离线、确定性、可重复刷取，不引入联网随机服务或新存档版本。

## 掉落规则

| 难度 | 普通房 | 精英房（Boss 前一房） | Boss 房 |
| --- | --- | --- | --- |
| 普通 | 非套装稀有 | 非套装稀有 | 副本池史诗 |
| 冒险 | 非套装稀有 | 副本池史诗 | 副本池史诗 |
| 勇士 | 副本池史诗 | 副本池史诗 | 副本池神话 |

- “副本池”严格等于 `DungeonDefinition.lootSetIds`。
- 非套装填充掉落只从 `rarity === "rare" && setId === undefined` 的目录装备中选择。
- 史诗/神话定向掉落只从对应稀有度且 `setId` 属于当前副本池的装备中选择。
- 选择完全确定：优先玩家持有数量最少的候选目录装备，再按房间与难度稳定偏移选择并列项。完成一轮候选池前不会因纯随机重复而卡住。
- 琉璃熔炉有五房；除最后两房按精英/Boss 规则外，其余房间均按普通房规则。

## 奖励倍率

- 经验只使用既有难度奖励倍率。
- 金币、玄铁尘和弧光碎片先使用既有难度倍率，再乘 `1 + goldFind / 100`，最终只四舍五入一次。
- `goldFind` 取当前 `CombatRun.combatProfile.stats.goldFind`，负值按 0 处理。
- 消耗品和装备身份不受数值倍率重复放大。

## 结算展示

- `AppViewModel.lastLoot` 保存最近一次已应用的 `CombatLootEvent`，只存在运行时，不写入版本化角色存档。
- 全局 `loot-result-banner` 在战斗下一房和通关回城后均可见。
- 展示经验、金币、材料、消耗品；有装备时展示稀有度、名称、部位和套装名，无套装时明确标记“散件”。
- 根节点提供稳定 `data-*`：副本、房间、装备 ID、稀有度、套装 ID、金币和材料，供真实浏览器验收读取。
- 打开新的副本准备或正式进入副本时清除旧结算；普通导航不误删刚通关结果。

## 验收

1. 单元测试证明三种难度每个节点的稀有度矩阵及两个副本池隔离。
2. 单元测试证明已有装备影响确定性轮换，且 `goldFind` 只作用一次。
3. 集成/UI 测试证明房间与最终通关都保留具体结算。
4. 真实浏览器以键盘完成勇士难度首房，看到副本池史诗结算并在刷新后确认装备仍在离线存档。
