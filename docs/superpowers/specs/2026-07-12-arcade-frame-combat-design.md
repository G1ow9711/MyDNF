# 街机逐帧战斗美术设计

## 目标

战斗表现参考经典横版格斗游戏的逐帧方法，但人物、怪物、技能名和美术全部原创。角色不能再是整张静态立绘位移，也不能使用程序化积木模型。每个动作必须有明确的准备帧、接触帧、跟随帧和收招帧。

## 运行架构

- 每套图集为 4x4、16 帧、透明 PNG，角色比例和地面线固定。
- `CombatSpriteStage` 读取现有战斗状态，不复制伤害、命中和移动规则。
- DOM 每 50ms 重建后，帧号仍由战斗时间、位移差、攻击进度和受击状态确定，因此不会从第一帧重新播放。
- 图集全部解码成功后才设置 `data-frame-sprite-stage="ready"`。加载或解码失败时继续显示旧资源。
- 普通帧分区：待机 0-3，跑动 4-7，攻击 8-11，受击 12-13，倒地 14-15。

## 首批角色

- 焰拳卫：红黑短战袍、炉火拳甲。拳甲绘制在每一帧手臂姿态内，避免独立武器层漂移。
- 灰烬小妖：山海经氛围的灰黑鳞甲妖物，胸口炉火核心，头、肩、肘、爪、膝和尾巴逐帧变化。
- 琉璃剑士：白青战袍、琉璃长剑，拥有普通动作图集和独立技能图集。

## 流光幻剑阵

技能体验借鉴高速剑舞的节奏结构，不复用受版权保护的名称或素材。

1. 开式：压低重心、拔剑蓄势、第一道横斩。
2. 交叉：反手、斜斩、低扫、上挑和交叉连斩。
3. 终式：前冲、举剑蓄力、全身终结斩和跪姿接触帧。
4. 收式：控制回收和还剑站立。

人物帧、青色剑轨、同帧残影、目标命中火花、hitstop 和屏幕反馈必须来自同一命中阶段。

## 验收

- 真实键盘移动时帧号进入 4-7；轻击进入 8-11；受击和倒地使用独立帧。
- 怪物自然攻击时使用攻击帧，而不是整图缩放。
- 流光连式通过 Space 施放，开式、交叉和终式均能采集对应人物帧、剑轨、残影和既有命中特效。
- 桌面和移动视口无绿幕、洋红底、黑色矩形旧残影或角色与武器分离。
- 战斗场景占满可用宽度，HUD 不遮住主要角色动作。
