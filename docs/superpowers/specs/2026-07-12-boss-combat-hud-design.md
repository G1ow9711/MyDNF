# Boss战斗状态栏设计

## 目标

Boss房必须提供持续、清晰、可操作的战斗信息：真实生命值、当前阶段、霸体护甲、破防窗口和当前施法。所有数值直接读取战斗状态，不增加独立的假进度条。

## 信息层级

1. Boss名称与阶段：灰烬窑巷显示“饕餮监工”，琉璃炉心显示“琉璃监工”。
2. 生命条：当前HP、最大HP、百分比和十段刻度。
3. 护甲条：当前护甲、该阶段最大护甲和状态标签。
4. 施法条：当前技能名、蓄势/释放/收招阶段和真实动作进度。

## 状态规则

- 初始Boss护甲为80/80。
- 饕餮第三阶段甲胄回升为至少120/120。
- `armorBrokenUntilMs`有效时显示“破防窗口”和剩余时间。
- 护甲为0且无破防计时时显示“护甲耗尽”。
- 其余护甲有效状态显示“霸体护甲”。
- Boss死亡后移除战斗状态栏，保留既有死亡演出和结算流程。

## 布局

- 桌面端位于技能栏下方、战场上缘，不覆盖左侧角色状态和右侧连击计数。
- 移动端缩为两行，位于操作区下方、角色战斗区域上方。
- 使用全局横向状态条，不嵌套卡片。

## 验收

- 普通房不渲染Boss状态栏。
- Boss房显示正确名称、520/520 HP、80/80护甲和阶段一。
- 阶段三显示120最大护甲。
- 施法时显示真实技能、动作阶段和进度。
- 破防时状态和剩余时间与权威战斗时间一致。
