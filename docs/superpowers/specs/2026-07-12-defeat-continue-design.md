# 地下城倒地继续与复活流程设计

## 目标

角色被怪物击倒后保留当前战场，显示明确的继续挑战或返回城镇决策。使用复活令后在原房间原地恢复，怪物与进度不重置；没有复活令时仍能安全回城。

## 权威行为

- 沿用现有 `performAction({ type: "consume", consumableId: "revival-token" })`。
- 复活恢复 35% 最大生命，获得 1200ms 无敌，消耗一枚复活令，并清除 `failed` 与 `defeated`。
- 房间、敌人 HP、掉落历史和任务状态保持不变。
- 复活后的消耗品数量立即同步到角色状态和本地存档。
- 没有复活令时不得修改战斗或存档状态。

## 界面

- 倒地时显示覆盖战场的 `data-defeat-overlay="true"` 决策层。
- 显示当前复活令数量和是否可继续。
- “继续挑战”按钮使用现有 `data-consumable-id="revival-token"`，同时暴露 `data-defeat-revive="true"`。
- 显示快捷键 `2`；“返回城镇”按钮使用现有 `data-mode="town"`。
- 无币时禁用继续按钮，显示 `data-defeat-reason="no-revival-token"`，回城按钮保持可用。
- 覆盖层出现时技能栏仍在背景中但不可执行，战场模型保持可见。

## 验收

1. App 测试证明真实失败运行通过现有 consumable action 恢复、无敌、扣一枚币且敌人状态不变。
2. UI 测试覆盖有币/无币两种覆盖层、按钮和原因钩子。
3. 真实浏览器使用低防御职业进入勇士难度，被自然怪物技能击倒，观察覆盖层，然后点击真实继续按钮。
4. 复活后断言 HP 大于零、无敌激活、覆盖层消失、敌人仍活着、复活令数量和 `localStorage` 同步减少一枚。
