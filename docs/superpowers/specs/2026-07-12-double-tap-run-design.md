# 双击方向奔跑与奔跑攻击设计

## 目标

补齐 DNF 风格键盘移动：短时间内连续按两次同一方向并保持第二次按键，角色进入奔跑；奔跑中按轻击直接复用现有 `dash-light` 奔跑攻击。

## 输入规则

- 支持上、下、左、右四个方向，双击窗口为 280ms。
- 第一次按下后必须发生松键，第二次按下才可进入奔跑；系统自动重复的 `keydown` 不算第二次输入。
- 奔跑持续到主方向松开、窗口失效场景切换、浏览器失焦或角色被战斗状态锁定。
- 按住 Shift 仍可进入冲刺，作为兼容和无障碍辅助输入。
- 方向指令缓冲继续去重相同方向，不把双击奔跑误识别为技能指令。

## 战斗衔接

- 双击奔跑只向现有 `CombatInput.dash` 提供输入来源，不修改速度、碰撞或命中规则。
- 奔跑形成现有 `dashAttackReadyUntilMs` 窗口；窗口内真实按下 X/J，进入现有 `dash-light` 位移、90ms 命中帧、受击反应和特效。
- 图集奔跑帧继续由位置变化驱动；双击来源额外显示脚底扬尘/速度线，不覆盖技能特效。

## 验收

- 单次方向点按不进入奔跑。
- 280ms 内同方向二次按下并保持，真实浏览器 DOM 暴露 `double-tap` 来源，位移速度和奔跑攻击准备窗有效。
- 保持第二次方向时按 X，真实进入 `dash-light` 并在命中帧伤害目标。
- 松开方向、失焦和离开战斗都清空双击奔跑状态。
