# 地下城通关评级与结果流程设计

## 目标

复现 DNF 地下城 Boss 结算后的独立结果阶段。玩家必须先看到通关评级、战斗统计、装备掉落和评级奖励，再通过真实按键或按钮返回城镇；不再从完成门直接跳回城镇。

## 权威数据

- `finishRoom()` 保留完整 `CombatRun.events` 并追加 `room-cleared`，时间轴继续使用单调递增的 `elapsedMs`。
- 旧事件不会重新播放：现有渲染器只显示处于事件 VFX 时间窗内的记录。
- 评级只读取最终运行的 `hit`、`miss`、`player-hit` 和通关时间，不写入角色存档。
- 掉落和评级奖励在进入结果模式前立即应用并自动保存。刷新结果页时安全恢复到城镇，奖励不会重复结算。

## 评分

总分限制在 0–10000：

- 基础分：1400，使没有任何攻击记录的异常通关落在 C 级，而不是仅凭未受击获得高评级。
- 连击分：`min(2400, maxCombo * 240)`。
- 命中分：`round(1800 * hits / max(1, hits + misses))`。
- 生存分：`max(0, 1800 - playerHitCount * 180)`。
- 时间分：以 `dungeon.rooms * 45000ms` 为目标，按 `round(1600 * (1 - clearTime / (targetTime * 1.5)))` 计算并限制在 0–1600。
- 暴击分：`min(600, criticalHitCount * 120)`。

评级阈值：C < 3500，B < 5000，A < 6500，S < 7800，SS < 9000，SSS ≥ 9000。

## 评级奖励

| 评级 | 金币 | 玄铁尘 |
| --- | ---: | ---: |
| C | 0 | 0 |
| B | 60 | 1 |
| A | 120 | 2 |
| S | 220 | 4 |
| SS | 360 | 6 |
| SSS | 520 | 10 |

评级奖励与最终房间掉落分开展示和应用，不修改 `CombatLootEvent` 的基础奖励语义。

## 运行时模型

- 新增 `AppMode = "dungeon-result"`。
- `DungeonClearResult` 包含副本、难度、评级、总分、评分明细、用时、命中/未命中、最大连击、暴击、受击、总伤害、最终掉落和评级奖励。
- `AppViewModel.dungeonResult` 仅存在于当前运行时。
- `confirmDungeonResult` 清除结果并切换到城镇 BGM；它不再次修改角色状态。

## 展示与控制

- 结果模式隐藏顶部系统导航，使用全屏单列结果界面。
- 第一视觉信号为大型 C–SSS 评级；同时显示分数、用时、命中率、最大连击、暴击、受击和伤害。
- 奖励区显示最终装备、基础金币/材料/消耗品及独立评级奖励。
- 挂载按钮使用 `data-confirm-dungeon-result`；Enter、Space 均确认，其他城镇快捷键不生效。
- 根节点暴露稳定的副本、难度、评级、分数、统计和奖励 `data-*`，供真实浏览器验收。

## 验收

1. 纯函数测试覆盖评分公式、评级边界、零攻击记录和奖励映射。
2. 战斗测试证明历史事件跨房保留，但旧命中特效不会在下一房重新显示。
3. App 测试证明 Boss 完成后停在结果模式，奖励只结算一次，确认后回城。
4. UI 测试证明评级、统计、掉落、奖励与响应式样式完整。
5. 真实键盘完成完整灰窑巷，必须先观察结果模式及存档奖励，再用 Enter 返回城镇。
