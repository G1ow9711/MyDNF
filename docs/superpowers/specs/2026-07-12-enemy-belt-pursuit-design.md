# 怪物同轴追击与攻击距离决策设计

日期：2026-07-12

## 目标

让怪物具备 DNF 式横版房间追击：主动靠近玩家、修正 Y 轴站位、进入招式有效距离后攻击，不再固定站位周期性挥空。

## 行为约束

- 普通怪、精英和 Boss 使用不同追击速度，均显著慢于玩家步行，保留走位与拉扯空间。
- 怪物按下一招的射程、冲锋距离或牵引距离决定接敌距离；召唤类招式不要求接近。
- 多只怪物使用错开的 Y 轴接敌槽位，避免完全堆叠成一个贴图。
- 攻击蓄力/恢复、受击硬直、控制、浮空、倒地、死亡和战斗 hitstop 期间禁止常态追击。
- 追击位置属于权威战斗状态，玩家技能的动态命中检测必须读取移动后的真实位置。

## 动作表现

- `CombatEnemy.aiState` 暴露 `idle` / `approach`，UI 输出稳定的 `data-enemy-ai-state`。
- 追击时位图模型播放独立步行动作，帧图舞台切换 run 帧；进入攻击后立即由招式动作接管。
- 怪物根节点平滑跟随权威坐标，不改变攻击、击退和死亡时间线。

## 验收

- 核心测试证明远距怪物 X/Y 坐标持续接近、进入有效距离后才开始攻击。
- 核心测试证明硬直、控制、浮空、倒地和攻击中均不发生常态追击。
- 真实浏览器在不移动玩家的情况下记录怪物 `approach -> attack`、坐标变化、run 帧和非空模型动画。
- 既有普通连段、剑舞、Boss 清场、掉落和存档流程保持通过。
