# 怪物死亡动作与灰烬消散设计

日期：2026-07-12

## 目标

把永久压扁的死亡贴图改为 DNF 风格的完整死亡收尾：致死接触、倒地、短暂停留、灰烬消散、从战斗层移除，最终只留下地面战利品。

## 权威时间

- `CombatEnemy.defeatedAtMs` 在 HP 首次从正数变为 0 时记录真实命中时间，后续不能覆盖。
- 普通、延迟技能和反射三条伤害路径都必须记录。
- 死亡演员总显示 1000 ms：0-180 ms 致死受击，180-520 ms 倒地，520-1000 ms 灰烬消散，之后不再渲染。
- 房间清空后仅继续推进尚未结束的死亡表现时间，最后一只怪消散后恢复停止空闲战斗时钟。
- 敌人仍保留在 `CombatRun.enemies` 供房门、统计和掉落结算使用；只从视觉演员层移除。

## 动作与视觉

- 帧舞台使用 12/14/15 对应致死受击、倒地、消散，不再按全局时间永久循环 14/15。
- 若致死攻击已有专用终结反应帧，前 180 ms 保留该接触姿态，再进入统一倒地和消散；例如 X 三连第三击保留帧 14 的击飞动作。
- 根节点根据死亡年龄计算负动画延迟，保证每次 DOM 重绘仍显示同一权威进度。
- 消散阶段降低饱和度和亮度，向上散出灰烬碎片并收缩地面阴影。
- 名牌和血条随尸体消散，掉落光柱保持独立可见。

## 验收

- 核心测试证明三类致死路径记录准确时间且不会被覆盖。
- 渲染测试证明三个死亡阶段与 1000 ms 后视觉移除，场景击败计数保持正确。
- 真实浏览器键盘清房，记录 death-impact/death-collapse/death-dissolve，随后确认怪物演员消失而地面装备仍在。
- 全量战斗、掉落、Boss、存档和帧动画回归通过。
